# 个人博客定制指南

这是一份详细的指南，帮助你将这个 Jekyll + Chirpy 主题的博客定制成你自己的个人博客。

## 必须修改的核心文件

### 1. `_config.yml` - 网站核心配置文件 ⭐⭐⭐

这是最重要的配置文件，包含了博客的所有基本信息。

**需要修改的主要内容：**

```yaml
# 第 17 行：博客标题
title: XK Blog  # 改成你的博客名称

# 第 19 行：副标题/座右铭
tagline: 天行健，君子以自强不息；地势坤，君子以厚德载物！  # 改成你的座右铭或标语

# 第 21-23 行：网站描述（用于 SEO）
description: >-
  分享关于数学、编程与科研的思考与记录。  # 改成描述你博客内容的简短介绍

# 第 27 行：网站 URL
url: "https://twahz.github.io"  # 改成你的 GitHub Pages 地址，如 https://你的用户名.github.io

# 第 30 行：GitHub 用户名
github:
  username: twahz  # 改成你的 GitHub 用户名

# 第 33 行：Twitter/X 用户名（可选）
twitter:
  username: GraphsGuru  # 改成你的 Twitter 用户名，或删除此部分

# 第 38-46 行：社交信息
social:
  name: Xingkun Song  # 改成你的真实姓名或昵称
  email: sxkmath@outlook.com  # 改成你的邮箱地址
  links:
    - https://x.com/GraphsGuru  # 改成你的 Twitter 主页
    - https://github.com/twahz  # 改成你的 GitHub 主页
    # 可以取消注释并添加更多社交链接

# 第 102 行：头像设置
avatar:  # 添加你的头像图片路径，如 /assets/img/avatar.jpg

# 第 9 行：语言设置
lang: zh-CN  # 如果需要英文博客，改为 en

# 第 12 行：时区
timezone: Asia/Shanghai  # 根据你的位置修改时区
```

**可选的高级配置：**

- **第 49-55 行：网站验证码**（用于 Google、Bing 等搜索引擎）
- **第 61-75 行：网站分析**（Google Analytics、GoatCounter 等）
- **第 111-131 行：评论系统**（Disqus、Utterances、Giscus）

### 2. `_data/contact.yml` - 联系方式配置 ⭐⭐

定义侧边栏底部显示的社交媒体图标和链接。

**需要修改的内容：**

```yaml
# 第 5 行：GitHub 链接
- type: github
  icon: "fab fa-github"
  url: "https://github.com/twahz"  # 改成你的 GitHub 主页

# 第 9 行：Twitter/X 链接
- type: x
  icon: "fa-brands fa-x-twitter"
  url: "https://x.com/GraphsGuru"  # 改成你的 Twitter 主页或删除

# 第 13 行：邮箱
- type: email
  icon: "fas fa-envelope"
  url: "mailto:sxkmath@outlook.com"  # 改成你的邮箱地址
```

**可选的联系方式：**
- 取消注释第 23-45 行来添加 Mastodon、LinkedIn、Stack Overflow、Bluesky、Reddit、Threads 等

### 3. `_tabs/about.md` - 关于页面 ⭐⭐

这是你的"关于我"页面，访客可以通过这个页面了解你。

**需要修改的内容：**

```markdown
# 第 6 行：页面描述
description: "你好，我是 XK"  # 改成你的简短自我介绍

# 第 9-22 行：整个关于页面的内容
你好，我是 **XK**，青椒一枚。

这个站点由 [GitHub Pages](https://pages.github.com/) 和 [Jekyll](https://jekyllrb.com/) 搭建...

# 全部替换成你自己的个人介绍、经历、兴趣等
```

### 4. `_posts/` 目录 - 博客文章 ⭐⭐⭐

这是你放置所有博客文章的地方。

**创建新文章的步骤：**

1. 在 `_posts/` 目录下创建新文件
2. 文件命名格式：`YYYY-MM-DD-标题.md`
   - 例如：`2024-11-03-my-first-post.md`

3. 文章开头需要包含 YAML Front Matter：

```markdown
---
title: 你的文章标题
date: 2024-11-03 10:00:00 +0800
categories: [分类1, 分类2]
tags: [标签1, 标签2, 标签3]
---

这里开始写你的文章内容...
```

**示例文章：**

```markdown
---
title: 我的第一篇博客
date: 2024-11-03 14:30:00 +0800
categories: [生活, 随笔]
tags: [开始, 个人博客]
---

# 欢迎来到我的博客

这是我的第一篇博客文章...
```

## 可选修改的文件

### 5. 头像图片

**第一步：创建 img 目录**

如果 `assets/img/` 目录不存在，需要先创建：

```bash
mkdir -p assets/img
```

**第二步：添加头像图片**

将你的头像图片文件复制到 `assets/img/` 目录下。

**第三步：在配置文件中设置**

然后在 `_config.yml` 中配置头像路径：

```yaml
avatar: /assets/img/你的头像文件名.jpg
```

建议尺寸：512x512 像素或更大，正方形

### 6. 网站图标 (Favicon)

替换 `assets/img/favicons/` 目录下的图标文件（如果存在该目录）

### 7. 其他标签页

`_tabs/` 目录下的其他文件：
- `archives.md` - 归档页（通常不需要修改）
- `categories.md` - 分类页（通常不需要修改）
- `tags.md` - 标签页（通常不需要修改）

这些文件通常保持默认即可，除非你想自定义页面标题或描述。

## 修改优先级总结

### 🔴 必须立即修改（否则会显示别人的信息）

1. `_config.yml` 中的个人信息（姓名、邮箱、用户名、URL）
2. `_data/contact.yml` 中的联系方式链接
3. `_tabs/about.md` 中的个人介绍

### 🟡 建议尽快修改

4. 添加自己的头像图片
5. 修改 `_config.yml` 中的标题、副标题、描述
6. 开始在 `_posts/` 中创建自己的文章

### 🟢 可选修改

7. 配置评论系统（如果需要）
8. 配置网站分析（如果需要）
9. 添加更多社交媒体链接
10. 自定义网站图标

## 本地预览博客

在修改完成后，你可以在本地预览博客效果：

```bash
# 安装依赖
bundle install

# 启动本地服务器
bundle exec jekyll serve

# 然后在浏览器访问 http://127.0.0.1:4000
```

## 部署到 GitHub Pages

1. 将所有修改提交到 GitHub 仓库
2. GitHub Actions 会自动构建和部署
3. 稍等几分钟后访问 `https://你的用户名.github.io`

## 注意事项

⚠️ **重要提醒：**

1. **不要删除** `.nojekyll` 文件
2. **不要修改** `_plugins/`, `index.html` 等文件（除非你知道自己在做什么）
3. **遵循文件命名规则**：博客文章必须使用 `YYYY-MM-DD-title.md` 格式
4. **提交前检查**：确保修改后的 `_config.yml` 语法正确（YAML 对缩进敏感）
5. **图片路径**：使用相对路径或以 `/` 开头的绝对路径
